# Project Brief — Contrast Security AI Transparency Page

**Project:** contrast-ai-transparency
**Type:** Static web application — public-facing trust/transparency page
**Owner:** Naomi Buckwalter, Sr. Director Product Security Engineering, Contrast Security
**Technical Lead:** Claude Code

---

## Target Deliverable

A publicly hosted, easily-updatable AI transparency webpage for Contrast Security,
modeled after HubSpot's Trust Center AI Model Cards page. Hosted on GitHub Pages
under the `contrastsecurity.dev` domain via the `Contrast-Security-OSS` GitHub org.

---

## Drivers

All of the following are active:
- Customer/prospect due diligence requests
- Regulatory/compliance pressure (EU AI Act, ISO 42001, NIST AI RMF)
- Proactive trust-building and market differentiation

## Target Audience

- Security/legal buyers (CISOs, procurement, legal — primary)
- Existing customers wanting AI data handling transparency
- Prospects evaluating Contrast vs. competitors
- Regulators/auditors requiring documented AI disclosures

---

## Technical Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Stack | Static site + YAML data file | Non-engineers update content via YAML; site auto-builds via GitHub Actions |
| Build step | `scripts/build.py` (YAML → JSON) | Simple, no framework dependency; runs in CI |
| Deployment | GitHub Actions → GitHub Pages | Native to target org; free hosting; version-controlled |
| Font | Poppins (Google Fonts) | Contrast brand digital alternative to Nohemi (not available on web) |
| CSS base | Adapted from `aiproc-poc/chrome-extension/popup.css` | Reuses prior brand implementation; consistent design system |
| Data format | `data/ai-usage.yaml` → `output/site/data.json` | YAML is human-readable; JSON is client-parseable without a build dependency |
